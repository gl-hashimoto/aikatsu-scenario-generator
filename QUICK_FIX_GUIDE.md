# 🚨 画像エラー緊急解決ガイド

## 現在の状況

会話履歴に大きな画像（2000ピクセル超）が含まれており、エラーが繰り返し発生している状態です。

## 即座にできる解決方法

### 方法1: 新しいチャットを開始する（最も簡単）

1. **Cursorの左側のサイドバーで「+ New Chat」ボタンをクリック**
2. 新しいチャットが開きます
3. この新しいチャットでは、問題のある画像は含まれていません
4. 必要に応じて、テキストのみで質問を再送信してください

### 方法2: 会話履歴を手動でクリアする

1. **Cursorの設定を開く**
2. **「Clear Chat History」または「Reset Chat」を選択**
3. 会話履歴がクリアされます

### 方法3: 画像を送信しないようにする

- 今後、画像を送信する必要がある場合は、**先に画像をリサイズ**してから送信してください
- 画像のサイズを確認する方法：
  - Mac: 画像を右クリック → 「情報を見る」→ サイズを確認
  - Windows: 画像を右クリック → 「プロパティ」→ サイズを確認

## 画像をリサイズする方法

### Macの場合

1. **プレビューアプリで画像を開く**
2. **「ツール」→「サイズを調整」**
3. **「幅」または「高さ」を2000ピクセル以下に設定**
4. **「OK」をクリック**
5. **「ファイル」→「保存」**

### Windowsの場合

1. **ペイントで画像を開く**
2. **「ホーム」タブ→「サイズ変更」**
3. **「ピクセル」を選択**
4. **「幅」または「高さ」を2000ピクセル以下に設定**
5. **「OK」をクリック**
6. **「ファイル」→「名前を付けて保存」**

### オンラインツールを使用する場合

1. **[TinyPNG](https://tinypng.com/)** - PNG/JPEG圧縮とリサイズ
2. **[Squoosh](https://squoosh.app/)** - 画像圧縮とリサイズ
3. **[ImageResizer](https://imageresizer.com/)** - 画像リサイズ

## 今後画像を送信する際の注意点

1. **画像のサイズを確認**: 送信前に幅と高さの両方が2000ピクセル以下であることを確認
2. **必要に応じてリサイズ**: 大きい場合は事前にリサイズ
3. **複数画像の場合**: すべての画像が2000ピクセル以下であることを確認

## 技術的な詳細

- **エラー内容**: `messages.74.content.4.image.source.base64.data: At least one of the image dimensions exceed max allowed size for many-image requests: 2000 pixels`
- **意味**: 74番目のメッセージの4番目のコンテンツアイテム（画像）のサイズが制限を超えている
- **制限**: 複数画像を含むリクエストでは、各画像の最大サイズは2000ピクセル

## それでも解決しない場合

1. **Cursorを再起動する**
2. **ブラウザのキャッシュをクリアする**（Web版を使用している場合）
3. **問題を報告する**: Cursorのサポートに連絡
