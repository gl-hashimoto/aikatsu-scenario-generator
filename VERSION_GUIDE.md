# バージョン管理ガイド

## バージョンアップ時の手順

新しい機能を追加したり、バグを修正したりした場合は、以下の手順でバージョンを更新してください。

### 1. バージョン番号を決定

**セマンティックバージョニング (Semantic Versioning)** を採用

```
MAJOR.MINOR.PATCH
例: 3.0.0
```

- **MAJOR（メジャー）**: 大きな機能追加・破壊的変更
  - 例: データ形式の変更、UIの大幅な刷新

- **MINOR（マイナー）**: 新機能追加（後方互換性あり）
  - 例: 新しいページの追加、新しい分析機能

- **PATCH（パッチ）**: バグ修正・小さな改善
  - 例: エラー修正、UI調整

### 2. app.pyのバージョン情報を更新

```python
# app.py の先頭部分
VERSION = "3.1.0"  # ← ここを更新
VERSION_DATE = "2025-11-15"  # ← 今日の日付に更新
```

### 3. CHANGELOG.mdに更新内容を記録

```markdown
## vX.X.X (YYYY-MM-DD)

### 新機能
- ✨ 追加した機能の説明

### 改善
- 🔧 改善した内容

### バグ修正
- 🐛 修正したバグ

### その他
- 📝 その他の変更
```

### 4. README.mdのバージョンを更新

```markdown
**Current Version: 3.1.0** (2025-11-15)
```

そして更新履歴セクションに追加。

## バージョンアップの例

### 例1: バグ修正（パッチバージョン）

```
3.0.0 → 3.0.1
```

**変更内容:**
- データ読み込み時のエラーを修正

**更新箇所:**
- `app.py`: VERSION = "3.0.1"
- `CHANGELOG.md`: v3.0.1のエントリを追加
- `README.md`: バージョン番号を更新

### 例2: 新機能追加（マイナーバージョン）

```
3.0.1 → 3.1.0
```

**変更内容:**
- CSVエクスポート機能を追加

**更新箇所:**
- `app.py`: VERSION = "3.1.0"
- `CHANGELOG.md`: v3.1.0のエントリを追加（新機能として記載）
- `README.md`: バージョン番号を更新、機能リストに追加

### 例3: 大きな変更（メジャーバージョン）

```
3.1.0 → 4.0.0
```

**変更内容:**
- OpenAI APIからClaude APIに変更（破壊的変更）
- 設定ファイルの形式を変更

**更新箇所:**
- `app.py`: VERSION = "4.0.0"
- `CHANGELOG.md`: v4.0.0のエントリを追加（破壊的変更を明記）
- `README.md`: バージョン番号を更新、移行ガイドを追加
